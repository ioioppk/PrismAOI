## 五、检测流程数据流图

```mermaid
flowchart TD
    START([开始检测]) --> TRIGGER["外部触发信号<br/>PLC / 光电传感器 / 软件触发"]
    TRIGGER --> LIGHT["光源切换<br/>ILightController.SetChannelAsync"]
    LIGHT --> CAPTURE["相机采集<br/>ICamera.CaptureAsync"]
    CAPTURE --> IMG["ImageData<br/>图像数据封装"]

    IMG --> ALIGN_STEP{"步骤类型?<br/>定位步骤"}
    ALIGN_STEP -->|是| ALIGN["模板匹配<br/>TemplateMatchProcessor"]
    ALIGN --> TRANSFORM["图像矫正<br/>AffineTransProcessor"]
    TRANSFORM --> INSPECT_STEP{"步骤类型?<br/>检测步骤"}
    ALIGN_STEP -->|否| INSPECT_STEP

    INSPECT_STEP -->|是| FILTER["预处理链<br/>灰度化→滤波→阈值"]
    FILTER --> BLOB["Blob分析<br/>BlobAnalysisProcessor"]
    BLOB --> JUDGE["结果判定<br/>数量/面积/位置"]

    INSPECT_STEP -->|否| MEASURE_STEP{"步骤类型?<br/>测量步骤"}
    MEASURE_STEP -->|是| MEASURE["卡尺测量<br/>MeasureProcessor"]
    MEASURE --> JUDGE

    MEASURE_STEP -->|否| COMM_STEP{"步骤类型?<br/>通讯步骤"}
    COMM_STEP -->|是| COMM["结果输出<br/>Modbus写寄存器 / WebAPI上报"]

    COMM_STEP -->|否| END_FLOW([检测完成])
    JUDGE --> SAVE{"NG?"}
    SAVE -->|是| SAVE_IMG["保存NG图像<br/>FileService"]
    SAVE -->|否| DISCARD["丢弃图像"]
    SAVE_IMG --> ALARM{"连续NG?"}
    ALARM -->|是| ALERT["触发报警<br/>AlarmService"]
    ALARM -->|否| COMM
    DISCARD --> COMM
    ALERT --> COMM
    COMM --> END_FLOW

    style START fill:#e8f5e9
    style END_FLOW fill:#e8f5e9
    style IMG fill:#e3f2fd
    style JUDGE fill:#fff3e0
    style ALERT fill:#ffebee
```

## 六、流程引擎执行时序图

```mermaid
sequenceDiagram
    actor User as 用户/PLC触发
    participant UI as WPF UI
    participant Exec as RecipeExecutor
    participant Factory as ProcessorFactory
    participant Proc as IImageProcessor
    participant Comm as ICommunicationAdapter
    participant Dev as IDevice(相机)

    User->>UI: 点击"执行"或外部触发
    UI->>Exec: ExecuteAsync(recipe, ct)

    loop 遍历每个步骤
        Exec->>Exec: 解析步骤类型

        alt 采集步骤
            Exec->>Dev: CaptureAsync()
            Dev-->>Exec: ImageData
        else 检测步骤
            loop 遍历子步骤
                Exec->>Factory: 获取算子实例
                Factory-->>Exec: IImageProcessor
                Exec->>Proc: ExecuteAsync(ctx, ct)
                Proc-->>Exec: ProcessResult
            end
        else 通讯步骤
            Exec->>Comm: SendAsync(data)
            Comm-->>Exec: CommunicationResult
        end

        Exec->>Exec: 检查 StopOnFailure
        alt 失败且需停止
            Exec->>Exec: break
        end
    end

    Exec-->>UI: RecipeResult
    UI->>UI: 显示检测结果
```

## 七、算子生命周期图

```mermaid
stateDiagram-v2
    [*] --> 已注册: 程序启动<br/>反射扫描所有<br/>IImageProcessor实现
    已注册 --> 工具箱展示: 按Category分组<br/>显示在工具箱面板
    工具箱展示 --> 已实例化: 用户拖入流程图<br/>或从DI容器解析
    已实例化 --> 参数配置: 用户在属性面板<br/>设置参数
    参数配置 --> 等待执行: 保存流程
    等待执行 --> 执行中: 触发执行
    执行中 --> 执行成功: 返回ProcessResult
    执行中 --> 执行失败: 异常/超时
    执行失败 --> 等待执行: 修复后重试
    执行失败 --> 跳过: StopOnFailure=false
    执行成功 --> 等待执行: 下一轮检测
    跳过 --> 等待执行: 下一轮检测
```

## 八、设备连接状态机

```mermaid
stateDiagram-v2

    连接中 --> 已连接: 连接成功

    重连中 --> 已连接: 重连成功
    重连中 --> 已断开: 重试耗尽→报警
    已连接 --> 已断开: DisconnectAsync()

## 十六、一句话总结

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  接口层：多留，一个不少   →   扩展性的根基                    │
│  实现层：少做，按需迭代   →   不浪费一行代码                  │
