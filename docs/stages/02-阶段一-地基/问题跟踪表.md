## 二、问题详情

### BUG-001：NModbus4 在 .NET 6 下兼容性问题

**类型**：Bug
**严重程度**：中
**优先级**：P2
**模块**：CommLib.Modbus
**状态**：✅ 已关闭

#### 问题描述
NModbus4 是 .NET Framework 时代的库，在 .NET 6 下通过兼容模式运行，存在兼容性风险。

#### 重现步骤
1. 在 .NET 6 项目中引用 NModbus4
2. 使用 Modbus TCP 功能
3. 可能出现运行时异常或不稳定

#### 根因分析
NModbus4 基于 .NET Framework 构建，未针对 .NET Core/.NET 6 进行适配。

#### 解决方案
改为原生 TcpClient 实现 Modbus TCP 协议，手动组包解析。
- 优点：完全可控，无第三方依赖，兼容性好
- 缺点：需要自己实现协议细节（约 400 行代码）

#### 验证结果
✅ ModbusTcpAdapter 原生实现已完成，编译通过，单元测试通过。

### BUG-002：Interlocked.Increment 不支持 ushort

**类型**：Bug
**严重程度**：低
**优先级**：P3
**模块**：CommLib.Modbus
**状态**：✅ 已关闭

#### 问题描述
`_transactionId` 变量定义为 `ushort` 类型，但 `Interlocked.Increment` 不支持 `ushort`，导致编译错误。

#### 重现步骤
```csharp
private ushort _transactionId;
Interlocked.Increment(ref _transactionId); // 编译错误
```

#### 根因分析
C# `Interlocked` 类的泛型方法不支持所有值类型，仅支持 `int`, `long`, `double`, `float`, `object` 等。

#### 解决方案
将 `_transactionId` 改为 `int` 类型，Modbus TCP 协议中 transaction ID 虽然是 16 位，但用 int 存储不影响功能。

#### 验证结果
✅ 编译通过，单元测试通过。

### BUG-003：Halcon 未安装时编译失败

**类型**：Bug
**严重程度**：高
**优先级**：P1
**模块**：ImageLib.HalconBridge
**状态**：✅ 已关闭

#### 问题描述
未安装 Halcon SDK 的开发机上，HalconDotNet.dll 不存在，导致 ImageLib.HalconBridge 和依赖它的项目编译失败。

#### 重现步骤
1. 在未安装 Halcon 的机器上克隆代码
2. 执行 `dotnet build`
3. 编译失败，提示找不到 HalconDotNet.dll

#### 根因分析
ImageLib.HalconBridge 直接引用 HalconDotNet，代码中直接使用 HObject 等类型。

#### 解决方案
使用 `HAS_HALCON` 编译常量 + 条件编译：
1. .csproj 中默认不定义 HAS_HALCON
2. 所有 Halcon 相关代码用 `#if HAS_HALCON` 包裹
3. 未定义时提供降级实现（抛出 NotSupportedException 或返回空对象）
4. 安装 Halcon 的开发者可以在 .csproj 或 Directory.Build.props 中启用

#### 验证结果
✅ 未安装 Halcon 环境下编译通过，所有单元测试通过。

### BUG-004：RecipeSerializer 枚举反序列化失败

**类型**：Bug
**严重程度**：中
**优先级**：P2
**模块**：WorkflowEngine.Core
**状态**：✅ 已关闭

#### 问题描述
使用 System.Text.Json 反序列化包含枚举的 JSON 时，默认按数字解析，而 demo_recipe.json 中枚举是字符串（如 "ImageAcquisition"），导致反序列化失败返回 null。

#### 重现步骤
```csharp
var json = "{\"type\":\"ImageAcquisition\"}";
var result = JsonSerializer.Deserialize<InspectionStep>(json);
// result.Type 默认值，未正确解析
```

#### 根因分析
System.Text.Json 默认按数值序列化/反序列化枚举，不支持字符串形式。

#### 解决方案
在 JsonSerializerOptions 中添加 `JsonStringEnumConverter`：
```csharp
Converters = { new JsonStringEnumConverter(JsonNamingPolicy.CamelCase) }
```

#### 验证结果
✅ 枚举正确序列化/反序列化，往返测试通过。

### DEBT-001：Dictionary<string, object> 反序列化后值为 JsonElement

**类型**：技术债务
**严重程度**：低
**优先级**：P3
**模块**：WorkflowEngine.Core
**状态**：🟡 待处理

#### 问题描述
`Dictionary<string, object>` 类型的参数在反序列化后，值的类型是 `JsonElement` 而不是原始类型（int, string, bool 等）。

#### 影响
- 算子中使用 `context.GetParam<double>()` 时需要做类型转换
- 目前通过 `Convert.ToDouble` 等方式处理，功能正常但不够优雅

#### 计划方案
引入自定义 JsonConverter，在反序列化时将 JsonElement 转换为对应 CLR 类型。
计划版本：V0.2.0
