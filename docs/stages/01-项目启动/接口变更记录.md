## 二、接口冻结说明

### 2.1 冻结范围
第一阶段冻结以下核心接口：
- IImageProcessor（图像处理算子接口）
- ICommunicationAdapter（通讯适配器接口）
- IConnectionMonitor / IStreamAdapter（通讯扩展接口）
- IDevice / ICamera / ILightController / IMotionController（设备接口）
- ILogger / IFileOperator / IConfigManager / IAlarmManager（系统接口）
- IRecipeExecutor（流程执行器接口）

### 2.2 冻结原则
1. **只增不删**：冻结期间只允许新增接口，不允许删除或修改已有接口签名
2. **向下兼容**：新增参数必须提供默认值
3. **变更审批**：接口变更需记录在本表，并经过技术负责人审批
4. **版本升级**：接口破坏性变更需升级主版本号

### 2.3 变更流程
1. 提出变更申请（描述问题、方案、影响）
2. 技术评审（评估兼容性、影响范围）
3. 更新接口定义规范文档
4. 更新本表记录
5. 通知所有相关开发人员

## 四、附录

### 4.1 变更类型定义
- **新增**：新增接口、类、方法、属性
- **修改**：修改接口签名、参数类型、返回值
- **删除**：删除接口、类、方法、属性
- **废弃**：标记为过时，保留兼容性
- **修复**：修复接口 Bug，不影响签名

### 4.2 版本号规则
- **主版本号**：不兼容的 API 修改
- **次版本号**：向下兼容的功能性新增
- **修订号**：向下兼容的问题修正
