# 阶段二：能用 — 阶段概述

**时间**：2026-07-19 ~ 2026-08-29（第 5-10 周，实际提前启动）
**目标**：流程编辑器完善、算子扩充、通讯协议扩充、设备管理、运行监控、集成测试

---

## 阶段目标

在阶段一地基之上，构建可用的视觉检测平台，具备完整的流程编辑、图像处理、设备通讯和运行监控能力。

## 各周计划

### 第 5 周：流程编辑器完善
- 新增/删除步骤（TreeView 右键菜单 + 按钮）
- 上移/下移步骤（拖拽排序或按钮）
- 步骤内子步骤编辑（DataGrid，增删改算子节点）
- 参数编辑控件（数值 NumericUpDown、字符串 TextBox、下拉 ComboBox）
- 保存/加载流程 JSON 文件
- 单步执行（执行到当前步骤停止，显示中间结果）

### 第 6 周：算子扩充至 10 个
- 均值滤波、中值滤波
- 高斯滤波、图像增强
- 模板匹配（基于形状）
- 仿射变换、极坐标变换
- 卡尺测量（边缘对）
- 阈值判定、区域数量判定

### 第 7 周：通讯协议扩充
- TCP Client 适配器
- TCP Server 适配器
- WebAPI 适配器（GET/POST，基于 HttpClient）
- 三菱 PLC 适配器（基于 HSLCommunication）
- 通讯节点接入流程（StepType = Communication）
- 通讯节点 + 图像处理联动测试

### 第 8 周：设备管理
- Basler 相机 SDK 封装（实现 ICamera）
- 相机实时采集 → 显示在界面上（连续采集模式）
- 光源控制器（串口/Modbus 控制，实现 ILightController）
- 设备管理面板（TreeView 列表 + 属性面板 + 连接/断开）
- 设备自动发现（扫描可用串口、网络设备）

### 第 9 周：运行监控
- 实时检测画面（当前图像 + 检测结果叠加）
- OK/NG 统计面板（计数、良率、趋势图）
- NG 图像自动保存（OK 不存，NG 存，节省磁盘）
- 简单报警（NG 连续 3 次 → 弹窗 + 日志 + 声音）
- 检测日志（每条检测记录：时间、结果、耗时、算子链）

### 第 10 周：集成测试 + 打磨
- 全模块串联测试（采集→处理→通讯→结果，完整流程）
- 异常处理补全（断线重连、采集失败、算子异常、超时）
- 日志完善（NLog 结构化日志，覆盖所有关键路径）
- 内存泄漏排查（dotMemory，修复对象池未归还问题）
- 启动速度优化（延迟加载模块、减少反射）

## 里程碑

- 第 8 周末：设备管理完成 → 打 Tag v0.2.0
- 第 10 周末：功能冻结 → 打 Tag v0.3.0
