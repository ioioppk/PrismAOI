# 提交规范

本项目采用 Conventional Commits 规范。每次提交应包含类型前缀和简短描述。

---

## 格式

```
<type>: <简短描述>
```

---

## 类型前缀

| 前缀 | 用途 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: 添加 BlobAnalysisProcessor` |
| `fix` | Bug 修复 | `fix: 修复串口超时未释放资源` |
| `refactor` | 重构（不改变功能） | `refactor: 提取 RecipeExecutor 公共逻辑` |
| `test` | 添加或修改测试 | `test: 补充 RecipeSerializer 边界测试` |
| `docs` | 文档变更 | `docs: 更新接口定义规范` |
| `chore` | 构建、依赖、工具 | `chore: 升级 NLog 到 5.x` |
| `style` | 格式调整 | `style: 统一缩进为 4 空格` |
| `perf` | 性能优化 | `perf: 图像传输使用对象池` |

---

## 多行提交

改动较大时可加正文，空一行分隔：

```
feat: 添加 SECS/GEM 通讯适配器

- 实现 HSMS 协议基础通信
- 支持 S1F1 Are You There 消息
- 添加连接状态机管理
```

---

## 分支命名

| 分支类型 | 命名 | 示例 |
|----------|------|------|
| 功能分支 | `feat/<描述>` | `feat/add-secs-gem` |
| 修复分支 | `fix/<描述>` | `fix/serial-timeout` |
| 重构分支 | `refactor/<描述>` | `refactor/comm-layer` |

---

## 提交粒度

- 一个提交只做一件事
- 构建可通过的提交（不提交编译失败的代码）
- 首次提交前先 run `dotnet build` 和 `dotnet test` 确认
