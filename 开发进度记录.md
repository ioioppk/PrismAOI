## 六、关键架构决策

1. **分层架构**: 严格遵循接口与实现分离，Core 层定义接口，Services/Operators 层实现
2. **DI 容器**: 流程引擎通过 `IServiceProvider` 动态解析算子实例
3. **算子发现**: 通过 `[Processor]` 特性自动发现和分组算子
4. **条件编译**: Halcon 模块通过 `HAS_HALCON` 条件编译，无许可时自动降级
5. **图像池化**: 通过 `ImagePool` 复用 `ImageData` 对象，减少 GC 压力
6. **流程序列化**: 基于 JSON 的流程定义，支持持久化存储和加载
